Using Docker

Docker can quickly let you build and run a new server with very minimal overhead. It can help to share projects by writing the server infrastructure into code files and sharing them using GIT.

Icon

Use GIT

Track all your projects changes using GIT. It will keep a complete history of all your committed changes. A second benefit is that your colleagues can download your work and thanks to the power of Docker, run them locally without having to configure their local machine

Icon

Use Jekyll

See how a static site builder like Jekyll can give you templating properties, letting you build something much more complex out of simpler components and giving you some database-like functionalities, without the overhead of an actual database.

Icon

Frontend Proxy

A frontend proxy can be run and act as a gateway to many projects that you would like to run side by side on the same machine. The benefits being that you don't need to 'port-hop' with various softwares running on different ports. You can also use real domain names locally, instead of always using localhost or 127.001.

Icon

Host your Jekyll website on GitHub

Github, with some configuration, will let you host your Jekyll project directly from their servers. You can even redirect a domain that you own to that project. This lets you use a custom domain name without hosting everything yourself.

Icon

Be Adventurous

This isn't all that you can do, this is just the beginning. Once you get used to these techniques, You could do something more advanced like develop a Node App. However GitHub doesn't support running those. You'll have to figure out where to host it yourself. But local development will run just fine.
